# ZIP 整合包功能 - 快速开始指南

## 🎯 功能概述

NOMM 现在支持 ZIP 格式的完整模组包导入导出。相比于简单的 JSON 格式，ZIP 包可以：

✅ 备份所有模组文件（不依赖网络仓库）
✅ 轻松分享给其他玩家
✅ 保留完整的模组结构和元数据
✅ 快速恢复和部署

---

## 🚀 快速使用

### 导出 ZIP 整合包（3步）

```
点击菜单 ⋮ → 选择 "Export Modpack (ZIP)" → 选择保存位置
```

**发生的情况：**
- 所有已启用的模组文件被打包
- 每个模组的 meta.json 被保留
- 生成单个 `modpack.zip` 文件

### 导入 ZIP 整合包（3步）

```
点击菜单 ⋮ → 选择 "Import Modpack (ZIP)" → 选择 ZIP 文件
```

**发生的情况：**
- ZIP 文件被解压到临时目录
- 所有模组文件复制到游戏目录
- 模组启用/禁用状态自动同步
- 临时文件自动清理

## 📦 ZIP 文件内容示例

```
mymodpack.zip
├── modpack.json
│   └── [
│       {"id": "BepInEx", "version": "5.4.20"},
│       {"id": "NuclearApiPlugin", "version": "1.0.0"}
│     ]
│
└── mods/
    ├── BepInEx/
    │   ├── BepInEx.dll
    │   ├── BepInExConfig.xml
    │   └── meta.json
    │
    └── NuclearApiPlugin/
        ├── NuclearApiPlugin.dll
        ├── config.json
        └── meta.json
```

## 🔍 常见问题

### Q: ZIP 和 JSON 格式有什么区别？

| 特性 | JSON | ZIP |
|------|------|-----|
| 文件大小 | 很小 | 较大 |
| 包含文件 | 仅 ID/版本 | 完整文件 |
| 需要网络 | 是 | 否 |
| 分享能力 | 差 | 好 |
| 备份用途 | 差 | 优秀 |

### Q: 导出 ZIP 需要多长时间？

- 小包（< 100MB）：几秒
- 中包（100-500MB）：十几秒
- 大包（> 500MB）：几分钟

### Q: ZIP 导入会覆盖现有模组吗？

不会。导入时：
- 新模组被复制
- 现有模组保持不变
- 您可以手动禁用不需要的模组

### Q: 导入失败了怎么办？

检查：
1. ZIP 文件是否完整（试试用解压工具打开）
2. 磁盘空间是否充足
3. 权限是否正确

### Q: 能修改 ZIP 内容吗？

可以！如果你有技术基础：
1. 用压缩工具打开 ZIP
2. 修改 meta.json 或删除不需要的模组
3. 重新压缩保存

## 🎓 高级技巧

### 技巧 1: 创建多个配置预设

```
gaming_modpack.zip     → 平衡、稳定
speedrun_modpack.zip   → 轻量、高性能
experimental.zip       → 最新、测试用
```

### 技巧 2: 版本控制

```
modpack_v1.0.zip      (备份 1)
modpack_v1.1.zip      (备份 2)
modpack_v2.0.zip      (备份 3)
```

### 技巧 3: 模组分类导出

虽然 NOMM 一次导出所有启用模组，但你可以：
1. 启用自己要的模组 → 导出
2. 禁用这些模组 → 启用其他模组 → 导出
3. 重复直到有所有需要的配置

## 🔐 安全建议

1. **定期备份**：每周导出一次当前配置
2. **验证文件**：导入后检查模组列表 
3. **隔离测试**：在新目录测试新的整合包
4. **保留原件**：不要删除原始 ZIP 文件

## 📞 故障排查快速表

| 问题 | 解决方案 |
|------|---------|
| ZIP 导出失败 | 检查磁盘空间，尝试重启应用 |
| ZIP 导入提示找不到文件 | 确保 ZIP 验证完直，文件名不含特殊字符 |
| 导入后模组未启用 | 手动启用，或检查 modpack.json |
| 临时文件未清理 | 手动删除 `/tmp/nomm_import_*` 文件夹 |
